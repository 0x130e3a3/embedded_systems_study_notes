# DDR 内存基础

## 术语表

### SDRAM
**Synchronous Dynamic Random Access Memory**（同步动态随机存取存储器）

DDR内存的前身，是一种同步工作的DRAM，在外部时钟信号上升沿和下降沿都触发数据传输。

#### SDRAM 内部结构

```
SDRAM Chip (颗粒)
├── Bank 0  ── 2D 阵列 (Row × Column)
├── Bank 1  ── 2D 阵列 (Row × Column)
├── Bank 2  ── 2D 阵列 (Row × Column)
└── Bank 3  ── 2D 阵列 (Row × Column)
```

- **Bank（存储单元组）**：DRAM芯片内部逻辑组织的基本单元。每个Bank是一个二维阵列（2D Array），由若干行（Row）和列（Column）交叉排列组成。芯片内通常有多个Bank（如4 Bank设计），通过Bank Address（BA[1:0]）选择。

- **Row（行）**：Bank阵列中的横向地址线，称为Wordline（字线）。访问时需先激活（Active）对应Row，才能读取该行数据。

- **Column（列）**：Bank阵列中的纵向地址线，称为Bitline（位线）。在Row激活后，通过列地址选中要读取的具体存储单元。

- **存储单元（Cell）**：Row与Column交叉点，每个Cell存储1bit数据。是SDRAM最小存储单元。注意这里容易混淆：Cell存储1bit，但**x4/x8/x16颗粒**指的是芯片对外的**数据IO位宽**——例如x8颗粒将内部8个相邻Cell编为一组，通过8条IO线并行传输8bit，而非每个Cell存8bit。

#### 读取流程
1. 通过Bank Address选定目标Bank
2. 通过Row Address激活（Active）对应Row
3. 通过Column Address选择列，读取该列数据
4. Precharge（预充电）关闭该Row，为下次访问做准备

#### 几个容易混淆的概念
- **Bank vs BANK**：小写bank指芯片内部逻辑Bank；大写BANK在模组语境中指多颗粒并联组成的存储组。
- **存储单元容量 vs 位宽**：存储单元容量指每个Cell存储1bit数据；位宽指数据总线宽度（x4/x8/x16颗粒即指其数据IO位宽），两者不同。
- **Row/Column寻址 vs DRAM容量**：Row和Column地址线位数决定寻址范围，共同决定芯片总容量。

#### SDRAM 容量计算

芯片总容量（bit数）公式：

```
单芯片容量 = Bank数 × Row数 × Column数 × 单Cell容量(1bit)
           = 2^(Bank Address位数) × 2^(Row Address位数) × 2^(Column Address位数)
```

**举例**：某SDRAM芯片 spec标注
- Bank Address：BA[1:0] → 2位 → 4个Bank
- Row Address：RA[12:0] → 13位 → 8192行
- Column Address：CA[9:0] → 10位 → 1024列
- 单Cell容量：1bit

```
容量 = 4 × 8192 × 1024 × 1bit = 32Mbit = 4MB
```

**两种等价算法：**

对于x8颗粒（IO位宽=8bit），每个Column地址对应1Byte（8个相邻Cell并联输出）：

```
总Byte数 = Bank数 × Row数 × Column数 × 每Column对应Byte数
        = 4 × 8192 × 1024 × 1 = 32MB
总bit数  = 32MB × 8 = 256Mbit
```

注意：这里的1指的是"每Column对应1Byte"，来自IO位宽÷8，而非额外乘以IO位宽再除8。

**DIMM模组容量** = 单芯片Byte容量 × 模组上颗粒数（多个颗粒并联提供模组位宽）

#### 常见DDR代次参数对比

| 代次 | 电压 | 单颗粒位宽 | 单颗粒最大容量 | 预取深度 |
|------|------|-----------|--------------|---------|
| DDR  | 2.5V | x4/x8/x16 | 256Mbit | 2n prefetch |
| DDR2 | 1.8V | x4/x8/x16 | 512Mbit | 4n prefetch |
| DDR3 | 1.5V | x4/x8/x16 | 2Gbit | 8n prefetch |
| DDR4 | 1.2V | x4/x8/x16 | 8Gbit | 8n prefetch |

**预取（Prefetch）**：芯片内部以2n/4n/8n burstsize读出数据，再一次性输出到外部数据总线，是DDR代次提升数据传输率的关键技术之一。

### SIMM
**Single In-line Memory Module**（单列直插内存模组）

一种老旧的内存条形式，数据位宽为16位或32位，需要成对使用。 SIMM模组有30针和72针两种规格，已被淘汰。

---

### DIMM
**Dual In-line Memory Module**（双列直插内存模组）

目前主流的内存条形式，相比SIMM触点分布在两侧，每组触点独立寻址。 常见位宽为64位（不含ECC）或72位（含ECC），可以单条使用。

---

### Bank

在DDR语境中，Bank包含两层含义：

- **Bank（存储单元组）**：DRAM芯片内部逻辑组织的基本单元，每个Bank是一个二维阵列，包含若干行（Row）和列（Column）。 访问时需先激活（Active）对应Row，再通过列地址读取数据。 芯片内部通常有多个Bank（如4 Bank设计），通过Bank Address区分。

- **BANK（存储库/颗粒）**：在多颗粒模组设计中，多个DRAM颗粒（Chip）并联组成一个BANK，共同承担数据总线的位宽。 模组通常由多个BANK构成（如两个BANK为一组），通过BANK地址线选择访问哪个BANK。

---

### 位宽（Bus Width）

数据总线的宽度，以bit为单位。DDR模组标准位宽为64位（不含ECC），ECC模组为72位。

- **模组位宽**：整个内存条对CPU数据总线的宽度
- **颗粒位宽**：每个DRAM芯片的数据输出宽度，常见有x4、x8、x16

### 为什么同容量芯片有不同位宽（x4 / x8 / x16）

在Bank结构不变（Bank数、Row数、Column数相同）的前提下，**Cell总数相同 → 容量相同**，但IO位宽可以不同。原因在于**内部阵列的列组织方式**不同：

| 颗粒位宽 | 每个Column对应的IO线数 | IO宽度 |
|---------|---------------------|-------|
| x4 | 4条IO线并行输出 | 4bit/次 |
| x8 | 8条IO线并行输出 | 8bit/次 |
| x16 | 16条IO线并行输出 | 16bit/次 |

Cell总数相同，但x16将内部2倍的Cell编为一组（列方向），每次读写的数据量更大。

#### 各颗粒位宽的优缺点

**x4 颗粒**
- 优：单芯片引脚少（地址线复用更复杂但数据线少），信号完整性好；适合高密度、大容量模组设计
- 缺：需要更多颗粒并联才能达到模组64位宽（需16颗x4并联），布局布线复杂

**x8 颗粒**
- 优：平衡之选，需8颗达成64位宽；颗粒数适中，PCB布局难度可接受；是目前最主流的规格
- 缺：性能/成本介于两者之间，无明显劣势

**x16 颗粒**
- 优：需最少颗粒达成模组位宽（仅需4颗x16并联）；地址线更少，封装成本低
- 缺：单芯片数据线达16条，在高速DDR4/DDR5下信号完整性挑战大；仅见于少数专用场景（如部分显存）

#### 实际应用举例

同样设计一条4GB DDR4 DIMM（64位宽）：
- 用 x4 颗粒：需要 **16颗**（64÷4=16）
- 用 x8 颗粒：需要 **8颗**（64÷8=8）
- 用 x16 颗粒：需要 **4颗**（64÷16=4）

模组总容量相同，但颗粒位宽决定模组设计上所需的颗粒数量、布局复杂度和信号完整性难度。

### 时许参数

#### SDRAM 引脚介绍

以经典SDRAM芯片（如Winbond W9820G6JH，8M×16×4 Bank）为例：

| 引脚类别 | 引脚名 | 方向 | 说明 |
|---------|--------|------|------|
| 时钟 | CLK | 输入 | 差分时钟输入，上升沿+下降沿触发 |
| 时钟使能 | CKE | 输入 | 时钟门控，高电平有效 |
| 片选 | CS# | 输入 | 低有效，Nvidia NTT配方禁止单独拉高 |
| Bank地址 | BA[1:0] | 输入 | 4 Bank选择 |
| 行地址 | A[12:0] | 输入 | 行地址线（同时用于预充电命令） |
| 列地址 | A[9:0] | 输入 | 列地址线 |
| 数据掩码 | DQM[1:0] | 输入 | 数据掩码，控制IO高阻 |
| 数据总线 | DQ[15:0] | 双向 | 16位数据IO |
| 电源 | VDD/VDDQ | 输入 | 3.3V/2.5V |
| 地 | VSS/VSSQ | 输入 | 地 |

#### 行有效时序（Row Active Timing）

```
        ___     ___     ___     ___
CLK  __|   |___|   |___|   |___|   |___
         |
         |<---- tRCD ---->|
         |               |
CS#  ____|               |________________
         |
         |<---- tRC ---->|
         |               
BA[1:0] _|_addr__________|________________
               |
A[12:0] _______|_________________________
               |<-Row Addr Valid->|

ACT  命令 --+
               |
               +--> Row 激活完成，数据可用于读取
```

- **tRCD (RAS to CAS Delay)**：从ACT命令到WE#可发出的最小周期，典型值15ns~20ns
- **tRC (Row Cycle Time)**：连续访问同一Bank同一Row的最小间隔，典型值60ns~70ns

#### 列读写时序（Column Read/Write Timing）

```
        ___     ___     ___     ___
CLK  __|   |___|   |___|   |___|   |___

ACT  ----+_________________________________
               |
               |<----- tRCD ---->|
               |
READ ----------+--> tCAC 读取延迟
               |
DOUT  _________/=========D=================
                     |<--- tAC --->|
               |<---- CL ---->|
```

- **CL (CAS Latency)**：从READ命令到数据输出的时钟周期数，是SDRAM重要性能指标
- **tAC (Access Time)**：从时钟边沿到数据输出的延迟时间（通常指相对于CLK的max delay）
- **tCAC (CAS Access Time)**：从CAS边沿到有效数据输出的延迟

#### CAS Latency（CAS延迟）

CL是DDR时代的关键时序参数，表示发出READ命令到数据返回之间的**时钟周期数**：

| CL等级 | 典型时钟频率 | tAC (max) | 说明 |
|--------|-------------|-----------|------|
| CL2 | 100MHz | 6ns | 早期SDRAM |
| CL2.5 | 133MHz | 5.6ns | PC133标准 |
| CL3 | 143MHz | 5.4ns | 降低频率换取稳定性 |

DDR时代CL以时钟周期为单位（如DDR-400 CL3表示3个时钟周期），实际延迟时间 = CL / 时钟频率。

#### 关键时序参数汇总

| 参数 | 含义 | 典型值范围 |
|------|------|-----------|
| tRCD | RAS to CAS Delay（行激活到列访问的延迟） | 15ns~20ns |
| tRP | Row Precharge Time（预充电时间） | 15ns~20ns |
| tRC | Row Cycle Time（行周期时间） | 60ns~70ns |
| tRAS | Row Active Strobe（行激活到预充电的延迟） | 40ns~50ns |
| CL | CAS Latency（列访问延迟） | 2~5 时钟周期 |
| tAC | Access Time（时钟边沿到数据输出的延迟） | 5ns~7ns |
| tDQS | DQS/DQ相对延时（DDR2+引入） | ±0.25ns |
