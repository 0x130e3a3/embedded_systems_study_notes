# DDR 内存学习笔记

## 一、基础概念

### 1.1 内存在计算机中的角色

内存是 CPU 与持久存储（硬盘/Flash）之间的**高速缓冲层**。CPU 直接访问内存获取指令和数据，而不直接访问硬盘——因为 DRAM 的读写延迟（~10ns 级）比 Flash（~100μs 级）快约一万倍。

内存在系统中扮演的角色可以用一句话概括：**临时仓库，断电即失**。

### 1.2 DRAM vs SRAM

| 特性 | DRAM | SRAM |
|------|------|------|
| 存储单元 | 1个电容 + 1个晶体管 | 6个晶体管（触发器） |
| 刷新 | 需要定期刷新（电容漏电） | 不需要刷新 |
| 密度 | 高（结构简单） | 低（6T 单元大） |
| 速度 | 较慢（需要行激活、预充电） | 快（直接寻址） |
| 成本 | 低 | 高 |
| 典型用途 | 主内存（DDR） | CPU 缓存（L1/L2/L3） |

**为什么用 DRAM 做主存**：容量大、成本低。虽然速度慢于 SRAM，但通过预取、多 Bank 交错等技术可以在系统层面弥补。

### 1.3 SDRAM：同步动态随机存取存储器

- **同步（Synchronous）**：内部所有操作以外部时钟为基准，命令发送和数据传输都与时钟边沿同步。区别于早期异步 DRAM（EDO DRAM）。
- **动态（Dynamic）**：存储单元由电容构成，需要定期刷新以防止数据丢失。
- **随机（Random）**：可以任意指定地址进行读写，不需要线性访问。

### 1.4 DDR 代次演进

| 代次 | 年份 | 电压 (VDD) | 预取深度 | 单芯片最大容量 | 典型速率 |
|------|------|-----------|---------|--------------|---------|
| DDR | 2000 | 2.5V | 2n | 256Mbit | 200-400 MT/s |
| DDR2 | 2003 | 1.8V | 4n | 512Mbit | 400-1066 MT/s |
| DDR3 | 2007 | 1.5V / 1.35V(L) | 8n | 8Gbit | 800-2133 MT/s |
| DDR4 | 2014 | 1.2V | 8n | 16Gbit | 2133-3200 MT/s |
| DDR5 | 2020 | 1.1V | 8n(双通道) | 64Gbit | 4800-6400+ MT/s |

**DDR3L**：DDR3 的低电压版本，VDD = 1.35V，向下兼容 1.5V。后缀 L 代表 Low Voltage。

**关键演进趋势**：
1. 电压逐代降低 → 功耗下降
2. 预取深度增加 → 内部吞吐提升
3. Bank 数量增加（DDR3: 8 Bank，DDR4: 16 Bank Group）
4. 传输速率持续提升

## 二、SDRAM 内部结构

### 2.1 存储单元（Cell）

每个存储单元（Cell）仅存储 **1bit** 数据，由 1 个电容 + 1 个晶体管构成。

- 电容充电 = 逻辑 1
- 电容放电 = 逻辑 0

**注意**：Cell 容量（1bit）与芯片位宽（x4/x8/x16）是两个不同的概念。位宽指的是芯片外部数据 IO 的并行位数，而非单个 Cell 能存多少。

### 2.2 Bank（存储阵列）：Row × Column 二维阵列

> 注：早期文献中称其为 "L-Bank"（逻辑 Bank）或 "逻辑 Bank"，用于与模组层面的 Rank 区分。现代 JEDEC 标准和数据手册中统一简称为 **Bank**。

SDRAM 芯片内部组织为一个或多个 **Bank**，每个 Bank 是一个二维阵列，由行（Row）和列（Column）交叉排列组成，每个交叉点是一个存储单元（Cell）。

![Bank存储阵列示意图](https://cdn-mineru.openxlab.org.cn/result/2026-04-25/4f566de7-f862-4892-9877-cf8337459b85/e5bfd0bbc33d90de539ca16f2be1c0b979dabdff3c2379c2ef2087b48cc8008a.jpg)

- **Row（行）**：又称 Wordline（字线）。访问时必须先通过 ACT 命令激活对应 Row，将该行所有 Cell 的数据读入行缓冲区（Sense Amplifier）。
- **Column（列）**：又称 Bitline（位线）。在 Row 激活后，通过列地址选中具体要读写的 Cell。

**为什么需要多个 Bank**：
单一 Bank 在同一时刻只能激活一行。当访问不同行时，必须先 Precharge（预充电）关闭当前行，再激活新行，这段时间内无法访问数据。多个 Bank 可以并行工作——一个 Bank 在预充电时，另一个 Bank 可以正常读写，大幅提高效率。

**各代次的 Bank 数量**：
- 早期 SDRAM：2 Bank
- DDR/DDR2：4 Bank
- DDR3：**8 Bank**
- DDR4：16 Bank（4 Group × 4 Bank）

### 2.3 寻址流程

访问 DDR 内存的标准流程（四步）：

1. **Bank Select** → 通过 BA[ ] 选择目标 Bank
2. **Activate** → 通过 RA[ ] 激活目标 Row（该行数据读入行缓冲区）
3. **Read/Write** → 通过 CA[ ] 选择目标 Column，进行读写
4. **Precharge** → 关闭 Row，恢复 Bank 到空闲状态

```
       CLK  ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
              └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
ACT  ──────────┐
               │<────── tRCD ──────>│
READ ──────────────────────────────┘
                                       │<── CL ──>│
DQ   ──────────────────────────────────┤══════════ D0 ══ D1 ══ ...
PRE  ──────────────────────────────────────────────────────┘
                                                       │<─ tRP ─>│
```

> **参考资料**：完整时序图见 `JESD79-3D_merged.md` 中的 Figure（Read Cycle Timing）、`4Gb_DDR3_merged.md` 中的 Timing Diagrams 章节。

### 2.4 容量计算公式与实例推演

**公式：**

```
芯片总容量（bit）= Bank数 × Row数 × 2^(Column地址位数) × 芯片位宽
```

**注意**：这里的"Column数"指的是**可寻址的位置数**，不是物理列数。每个位置同时输出等于芯片位宽的 bit 数。

实际物理结构是：每行有 `2^(Column地址位数) × 芯片位宽` 个物理 Cell 列，Column 地址选择其中一个"块"（块内所有 Cell 并行输出）。

```
容量（bit）= Bank数 × Row数 × Column位置数 × 位宽
```

**实例：** 某 DDR3 芯片（Micron MT41K256M16，4Gb）

| 参数 | 值 |
|------|-----|
| Bank 地址 | BA[2:0] → 3位 → 8 Bank |
| Row 地址 | RA[14:0] → 15位 → 32768行 |
| Column 地址 | CA[9:0] → 10位 → 1024个位置 |
| IO 位宽 | x16 |

```
总bit数 = 8 × 32768 × 1024 × 16 = 4Gbit
总Byte数 = 4Gbit / 8 = 512MB
```

物理上，每行实际有 1024 × 16 = 16384 个 Cell 列，Column 地址选其中 1 个块（16 列），并行输出 16bit。

### 2.5 易混淆概念辨析

| 混淆点 | 澄清 |
|--------|------|
| Cell 容量 vs 位宽 | Cell = 1bit；位宽 = 一次并行传输的位数（x4/x8/x16），详见 **3.2** |
| Row/Column 地址线位数 | 决定可寻址的行列数，不等于物理行列数 |
| "每 Column 对应多少 Byte" | x8 芯片：每 Column 对应 1Byte（8个 Cell 并行输出）；x16 芯片：每 Column 对应 2Byte |
| Bank 地址 (BA) | 芯片内部选择哪个 Bank，与模组的 Rank 无关 |

## 三、芯片位宽

### 3.1 什么是位宽（x4 / x8 / x16）

位宽指每个 DDR 芯片在一次传输周期内能提供的数据位数。常见的有三种：

| 芯片位宽 | 每次传输位数 | 典型应用场景 |
|---------|------------|------------|
| x4 | 4bit | 服务器 ECC 模组、高密度设计 |
| x8 | 8bit | 消费级最主流选择 |
| x16 | 16bit | 嵌入式/移动端、低引脚数场景 |

### 3.2 同容量下不同位宽的 Cell 组织方式差异

这是一个常见的混淆点：**两种计算中的"Column"概念不同**。

**两种等价的容量算法：**

| 算法 | 公式 | 说明 |
|------|------|------|
| 按物理列 | `Bank × Row × 物理列数 × 1bit` | 每个 Cell = 1bit |
| 按可寻址位置 | `Bank × Row × 2^(Column地址位数) × 位宽` | 每个位置输出位宽个 bit |

两种算法结果相同，因为：**物理列数 = 可寻址位置数 × 位宽**。

**实际硬件结构：**

```
行激活后，整行的 Cell 都读入 Sense Amplifier:

  ┌── 物理列 ──────────────────────────┐
  │ C C C C C C C C ... C C C C C C C │  ← x16 时每行实际有 16384 列 Cell
  └───┬────┬────┬────┬────┬─── ... ───┘
      │    │    │    │    │
   ┌──┴───┴┐┌──┴───┴┐┌──┴───┴┐
   │ 16个   ││ 16个   ││ 16个   │  ← 每 16 个 Cell 为一组
   │ 并行   ││ 并行   ││ 并行   │
   └───┬───┘└───┬───┘└───┬───┘
       │        │        │
   Column 地址 0   1   2   ...   ← 10位地址 = 1024个可选位置
                                   选哪个，哪个就输出 16bit
```

Column 地址（10位）只负责选择 1024 个"位置"，每个位置里有 16 个物理列，通过多路器并行输出 16bit。

**同容量不同位宽的芯片对比**（均为 4Gbit, 8 Bank, Row=15位）：

| | x4 | x8 | x16 |
|--|----|----|-----|
| 每行物理 Cell 列数 | 32768 | 32768 | 32768 |
| 每组 Cell 数（=位宽） | 4 | 8 | 16 |
| 可寻址位置数 | 8192 | 4096 | 2048 |
| Column 地址位数 | 13 | 12 | 11 |
| 总容量 | 8 × 32768 × 8192 × 4 = 4Gbit | 8 × 32768 × 4096 × 8 = 4Gbit | 8 × 32768 × 2048 × 16 = 4Gbit |

**关键理解**：在容量相同的前提下，位宽越宽，可寻址位置数越少，但每个位置并行输出的 bit 越多，总容量不变。

### 3.3 x4 vs x8 vs x16 的优缺点对比

| | x4 | x8 | x16 |
|--|----|----|----|
| 优点 | 信号完整性好 | 平衡之选 | PCB 上只需 1 颗，布线最简单 |
| 缺点 | 需要更多芯片并联 | 需要 2 颗 | 高速下信号完整性挑战大 |
| i.MX 6ULL (16-bit) 所需芯片数 | 4颗 | 2颗 | **1颗** |

### 3.4 为什么嵌入式常用 x16

嵌入式系统（如 i.MX 6ULL）的 DDR 控制器总线位宽通常是 **x16**。x16 芯片只需 **1 颗** 就能匹配控制器位宽，PCB 上只放一颗芯片，布线最简单、成本最低。

x4/x8 更多见于桌面 DIMM 模组设计。

## 四、嵌入式 DDR 硬件设计基础

### 4.1 两个核心概念：芯片 vs Rank

这是最容易混淆的地方，先区分清楚：

| 概念 | 定义 | 层级 |
|------|------|------|
| **芯片（Chip/Device）** | 一颗物理的 DDR 存储芯片（PCB 上的黑色方块） | 物理 |
| **Rank** | 一组芯片的集合，总位宽 = SoC DDR 总线位宽 | 逻辑 |

**关键区别：1 个 Rank 不一定等于 1 颗芯片。** 一个 Rank 需要几颗芯片，取决于单颗芯片的位宽和 SoC 总线位宽：

```
桌面 DIMM（64-bit 总线）:
  ┌─x8─┬─x8─┬─x8─┬─x8─┬─x8─┬─x8─┬─x8─┬─x8─┐
  │ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ 7  │ 8  │  ← 这 8 颗 = 1 个 Rank
  └────┴────┴────┴────┴────┴────┴────┴────┘
  8 × 8 = 64bit ✓

嵌入式 PCB（x16 总线，如 i.MX 6ULL）:
  ┌── x16 ──┐
  │  第 1 颗  │  ← 这 1 颗 = 1 个 Rank
  └─────────┘
  1 × 16 = 16bit ✓
```

嵌入式恰好 1 颗 = 1 个 Rank，是因为 1 颗 x16 的位宽已经够了 SoC 总线位宽。

### 4.2 SoC DDR 总线位宽 vs 桌面 DIMM

**嵌入式系统不使用 DIMM 模组**，DDR 芯片直接焊在 PCB 上。桌面 DIMM 的 64-bit 标准与嵌入式无关。

| | 桌面/服务器 PC | 嵌入式 SoC（如 i.MX 6ULL） |
|--|--------------|-------------------------|
| CPU 数据总线 | 64-bit | 32-bit |
| 内存形式 | DIMM 模组（可插拔） | PCB 直焊芯片 |
| DDR 总线位宽 | 64-bit（固定标准） | 由 SoC 决定（i.MX 6ULL = x16） |

**嵌入式系统的关键规则：** 芯片并联后的总位宽必须匹配 SoC 的 DDR 控制器接口位宽。

以 i.MX 6ULL 为例，MMDC DDR 接口固定为 **x16**：

| 单芯片位宽 | 需要芯片数 | 总位宽 | 说明 |
|---------|---------|--------|------|
| x16 | 1颗 | 16bit | 直接匹配，最简单 |
| x8 | 2颗 | 16bit | 需要2颗并联 |
| x4 | 4颗 | 16bit | 需要4颗并联 |

i.MX 6ULL 通常使用 1 颗 x16 芯片；若需要更大容量，可通过 2 颗 x16 芯片 + CS0/CS1 片选扩展容量（变为 Dual Rank），而非增加位宽。

### 4.3 Rank 与片选（CS）

Rank 通过片选信号（CS#）切换。i.MX 6ULL 支持 2 个片选（CS0 和 CS1）：

- **Single Rank（1R）**：1 颗 x16 芯片接 CS0，容量 = 单芯片容量
- **Dual Rank（2R）**：2 颗 x16 芯片分别接 CS0 和 CS1，容量 = 2 × 单芯片容量

```
Single Rank:                          Dual Rank:
  SoC ── CS0 ──→ [x16 芯片]            SoC ── CS0 ──→ [x16 芯片 #1]
                                             CS1 ──→ [x16 芯片 #2]
  容量 = 1颗的容量                     容量 = 2颗的容量
  位宽 = 16bit                         位宽 = 16bit（每次只选中一颗）
```

> 注：早期文献中称 Rank 为 P-Bank（Physical Bank），这是过时的叫法。现代 DDR 规范统一使用 Rank。

确定 Rank 数的方法：`Rank数 = 芯片总容量 / 单芯片容量`

### 4.4 SIMM 与 DIMM（了解即可）

SIMM 和 DIMM 是桌面/服务器的内存模组形式，嵌入式系统不使用。了解即可：

| 类型 | 全称 | 位宽 | 现状 |
|------|------|------|------|
| SIMM | Single In-line Memory Module | 16bit/32bit | 已淘汰 |
| DIMM | Dual In-line Memory Module | 64bit（标准） | 桌面/服务器主流 |

DIMM 的 64-bit 位宽来自桌面 CPU 的数据总线宽度，与嵌入式 SoC 无关。

## 五、引脚与命令集

### 5.1 引脚分类

以 DDR3 芯片为例：

#### 时钟引脚

| 引脚 | 说明 |
|------|------|
| CK / CK# | 差分时钟输入，命令和地址在 CK 上升沿采样 |

#### 控制引脚

| 引脚 | 说明 |
|------|------|
| CS# | 片选，低有效。为高时忽略命令 |
| CKE | 时钟使能，高有效。为低时进入 Power Down / Self Refresh |
| RAS# / CAS# / WE# | 命令编码，与 CS# 组合决定操作类型 |
| ODT | On-Die Termination 控制，高有效 |
| RESET# | DDR3 新增复位引脚，低有效 |

#### 地址引脚

| 引脚 | 说明 |
|------|------|
| BA[2:0] | Bank Address，选择目标 Bank（DDR3 为 3 位 = 8 Bank） |
| A[15:0] | 复用地址线，在不同命令阶段传输 Row 或 Column 地址 |

#### 数据引脚

| 引脚 | 说明 |
|------|------|
| DQ[ ] | 双向数据总线 |
| DQS / DQS# | 差分数据选通信号，读/写时与数据同步变化 |
| DM / DMI | 数据掩码，高时屏蔽对应 Byte 的写入 |

#### 电源引脚

| 引脚 | 说明 |
|------|------|
| VDD | 核心电源（DDR3: 1.5V，DDR3L: 1.35V） |
| VDDQ | IO 电源（同 VDD） |
| VREF | 参考电压 |
| VSS / VSSQ | 地 |
| ZQ | ZQ 校准引脚，外接 240Ω 精密电阻 |

### 5.2 DDR3 命令集

命令由 CS#、RAS#、CAS#、WE# 四信号组合决定：

| CS# | RAS# | CAS# | WE# | 命令 | 说明 |
|-----|------|------|-----|------|------|
| L | L | H | H | ACT | 激活 Bank + Row |
| L | H | L | H | READ | 读操作 |
| L | H | L | L | WRITE | 写操作 |
| L | H | H | L | PRE | 预充电（关闭 Bank） |
| L | H | H | H | REF | 自动刷新 |
| L | L | L | H | MRS | 模式寄存器设置 |
| L | L | L | L | MRS | 模式寄存器扩展设置 |
| H | X | X | X | DES | 空操作（Deselect） |
| L | H | X | H | NOP | 空操作 |

> X = 不关心（Don't care）

#### 关键命令详解

**ACT（Activate）**：激活指定 Bank 的 Row。发送 Row 地址（A[ ]）和 Bank 地址（BA[ ]）。

**READ**：发起读操作。发送 Column 地址（A[ ]）、Bank 地址（BA[ ]）。AP（A10）决定是否同时执行 Auto-Precharge。

**WRITE**：发起写操作。格式同 READ。

**PRE（Precharge）**：关闭已激活的 Row。A10=H 时预充电所有 Bank，A10=L 时仅预充电指定 Bank（通过 BA[ ]）。

**REF（Auto Refresh）**：自动刷新命令。DDR3 要求每 7.8μs 发送一次（典型值 tREFI）。

**MRS（Mode Register Set）**：配置模式寄存器。通过 BA[ ] 选择目标寄存器（MR0~MR7），通过 A[ ] 设置参数。

### 5.3 模式寄存器（Mode Register）

DDR3 有 8 个模式寄存器（MR0 ~ MR7），通过 MRS 命令配置：

#### MR0（模式寄存器 0）

| 位 | 名称 | 说明 |
|----|------|------|
| A0-A2 | CAS Latency (CL) 低位 | 5/6/7/.../14 |
| A3 | 写恢复 (Write Recovery) | |
| A4 | 突发类型 | 0=Sequential, 1=Interleaved |
| A5 | 突发长度低位 | |
| A6-A8 | CAS Latency 高位 | |
| A9 | 测试模式 | |
| A10 | 写恢复高位 | |
| A11 | DLL 复位 | 1=重置 DLL |
| A12 | 电源节省模式 | |

**常用 CL 值**：DDR3 常见 CL = 5/6/7/8/9/10/11

#### MR1（模式寄存器 1）

| 位 | 名称 | 说明 |
|----|------|------|
| A0 | 输出驱动强度 | |
| A1 | 禁用 DLL | |
| A6 | ODT 动态控制 | |
| A9 | 写均衡使能 | DDR3 新增 |

#### MR2（模式寄存器 2）

| 位 | 名称 | 说明 |
|----|------|------|
| A0-A2 | CAS Write Latency (CWL) | 5/6/7/8/9/10 |
| A5-A7 | 动态 ODT | |
| A9 | 自刷新温度范围 | |

#### MR3 ~ MR7

用于特殊功能：多用途寄存器（MPR）、自刷新控制等。

## 六、时序参数

### 6.1 行有效时序

#### tRCD（RAS to CAS Delay）

从 ACT 命令发出到 READ/WRITE 命令可发出的**最小间隔**。

```
ACT  ──────┐
           │<──── tRCD ────>│
READ ──────────────────────┘
```

典型值：12-15ns。DDR3-1600 CL11 中，tRCD = 13.75ns。

#### tRC（Row Cycle Time）

同一 Bank 中两次 ACT 命令之间的**最小间隔**。典型值：48-60ns。

```
ACT1 ─────────────────────────────────┐
          │<────────── tRC ──────────>│
ACT2 ─────────────────────────────────┘
```

#### tRAS（Row Active Strobe）

从 ACT 命令到同一 Bank 发出 PRE 命令的**最小间隔**。即 Row 必须保持激活的最短时间。

```
ACT  ──────┐
           │<─────── tRAS ───────>│
PRE  ─────────────────────────────┘
```

典型值：33-40ns。

关系：`tRC ≥ tRAS + tRP`

### 6.2 列读写时序：CL（CAS Latency）

CL 是最关键的时序参数之一，表示从 READ 命令发出到**第一个数据出现在 DQ 上**的时钟周期数。

```
READ ────┐
         │<────── CL 周期 ──────>│
DQ   ────────────────────────────┤══ D0 ══┤══ D1 ══┤
```

**注意**：CL 的单位是**时钟周期**，不是纳秒。

```
实际延迟（ns）= CL / 时钟频率（MHz）
```

例如：DDR3-1600（时钟 800MHz）CL11
```
实际延迟 = 11 / 800MHz = 13.75ns
```

#### tAC（Access Time）

从时钟边沿到数据实际出现在 DQ 引脚上的**绝对延迟**（ns）。CL 保证的是周期数，tAC 保证的是物理延迟上限。

### 6.3 预充电时序：tRP（Row Precharge Time）

从 PRE 命令发出到新的 ACT 命令可发出的**最小间隔**。

```
PRE  ──────┐
           │<──── tRP ────>│
ACT  ──────────────────────┘
```

典型值：12-15ns。

### 6.4 刷新时序

| 参数 | 含义 | DDR3 典型值 |
|------|------|-----------|
| tREFI | 两次刷新命令之间的间隔 | 7.8μs（正常），3.9μs（高温） |
| tRFC | 刷新命令持续时间 | 110-300ns（取决于容量） |

**注意**：tRFC 是 DDR 中最大的时序参数之一，刷新期间所有 Bank 不可访问。

### 6.5 DQS 相关时序（DDR2+ 引入）

| 参数 | 含义 |
|------|------|
| tDQSCK | DQS 相对于 CK 的延迟 |
| tQH | DQ 相对于 DQS 的保持时间 |

DQS（Data Strobe）是 DDR 引入的数据选通信号，与数据同步变化，用于接收端精确定位数据的有效窗口。

### 6.6 完整读时序

> 完整读时序图请参考 JEDEC JESD79-3D 标准文档中的 Figure（Read Cycle Timing）。

时序步骤：
1. 发出 ACT 命令，激活 Row
2. 经过 tRCD 后，发出 READ 命令
3. 经过 CL 周期后，第一个数据出现在 DQ 上
4. 发出 PRE 命令（关闭 Row）
5. 经过 tRP 后，可以激活新 Row

### 6.7 完整写时序

```
       CLK  ──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
              └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘
ACT  ──────────┐
               │<────── tRCD ──────>│
WRITE ─────────────────────────────┘
               │<── CWL ──>│
DQ   ───────────────────────┤══ D0 ══┤══ D1 ══┤══ ... ══┤
DQS  ───────────────────────┘↑______↑______↑______↑______↑
                             数据与 DQS 边沿同步
PRE  ────────────────────────────────────────────────────┘
                                                       │<─ tRP ─>│
```

**CWL（CAS Write Latency）**：DDR3 新增参数，表示 WRITE 命令到第一个写数据出现在 DQ 上的时钟周期数。不同于读操作的 CL，写操作不需要等待数据从阵列中读出，因此 CWL 通常 ≤ CL。

**写操作特点**：
- 数据通过 DQS 选通信号同步写入，DQS 在写操作时由 SoC 发出
- 写突发长度与预取深度匹配（DDR3 = BL8）
- 写入后需要等待 tWR（Write Recovery Time）才能执行 PRE

> **参考资料**：完整写时序图见 `JESD79-3D_merged.md` 中的 Figure（Write Cycle Timing）、`4Gb_DDR3_merged.md` 中的 Timing Diagrams 章节。

### 6.8 时序参数总结表

| 参数 | 全称 | 含义 | DDR3-1600 典型值 |
|------|------|------|-----------------|
| tRCD | RAS to CAS Delay | ACT → READ/WRITE 最小间隔 | 13.75ns |
| tRP | Row Precharge Time | PRE → ACT 最小间隔 | 13.75ns |
| tRC | Row Cycle Time | ACT → ACT 最小间隔（同Bank） | 48.75ns |
| tRAS | Row Active Strobe | ACT → PRE 最小间隔 | 35ns |
| CL | CAS Latency | READ → 数据输出的时钟周期数 | 11 cycles |
| CWL | CAS Write Latency | WRITE → 数据输入的时钟周期数 | 8 cycles |
| tAC | Access Time | 时钟边沿 → 数据输出的绝对延迟 | 5-6ns |
| tREFI | Refresh Interval | 刷新命令间隔 | 7.8μs |
| tRFC | Refresh Cycle Time | 刷新命令持续时间 | 160ns |
| tXP | Exit Power Down | 退出 Power Down 的最小间隔 | 6ns |
| tDQS | DQS-DQ Skew | DQS 与 DQ 之间的偏移 | ±0.25ns |

## 七、预取与突发传输

### 7.1 预取（Prefetch）机制

预取是 DDR 提升数据传输率的核心技术。其思想是：**芯片内部以较慢的频率并行读出多位数据，再以较高的频率一次性输出到外部总线**。

以 8n 预取为例：内部核心以 100MHz 频率并行读出 8 个数据字，外部 IO 以 800MHz 频率分 8 个时钟周期依次输出。

```
DDR3 8n 预取示意（内部 100MHz → 外部 800MHz）:

内部核心 (100MHz):
  ┌───────┐ ┌───────┐ ┌───────┐
  │ D0-D7 │ │ D8-D15│ │D16-D23│  ← 每周期并行读 8 个数据字
  └───┬───┘ └───┬───┘ └───┬───┘
      │         │         │
      ▼         ▼         ▼
外部 IO (800MHz):
  ┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐┌─┐
  │0││1││2││3││4││5││6││7│ ... ← 分 8 个时钟依次输出
  └─┘└─┘└─┘└─┘└─┘└─┘└─┘└─┘
  ↑                           ↑
  1 个内部周期 = 8 个外部周期
```

| 代次 | 预取深度 | 内部核心频率 | 外部 IO 频率 | 关系 |
|------|---------|-------------|-------------|------|
| SDR | 1n | 100MHz | 100MHz | 1:1 |
| DDR | 2n | 100MHz | 200MHz (100×2) | 1:2 |
| DDR2 | 4n | 100MHz | 400MHz (100×4) | 1:4 |
| DDR3 | 8n | 100MHz | 800MHz (100×8) | 1:8 |
| DDR4 | 8n | 100MHz | 800MHz (100×8) | 1:8 |

**关键理解**：DDR3 在内部核心频率不变（100MHz）的情况下，通过 8n 预取实现了 800MT/s 的外部传输率。

> **参考资料**：预取架构详解见 `MinerU_markdown_SM41J256M16M_*.md` 中的"预取架构"章节。

### 7.2 突发长度（Burst Length）

突发长度指一次 READ/WRITE 命令传输的数据字数。

| 模式 | Burst Length | 说明 |
|------|-------------|------|
| DDR3 固定 | BL8 | 每次传输 8 个数据字（x16 时 = 8×16bit） |
| DDR3 可选 | BC4/BL8 | 通过 MR0 配置，或 A12 位实现 on-the-fly 切换 |
| LPDDR2 | BL4 | 低功耗模式 |

**为什么 DDR3 默认 BL8**：因为 8n 预取一次读出 8 个数据字，BL8 刚好匹配预取深度，效率最高。

### 7.3 突发类型

| 类型 | 说明 |
|------|------|
| Sequential | 地址连续递增（最常见） |
| Interleaved | 地址按特定交织模式排列（用于特殊应用场景） |

**Sequential 示例**（BL8，起始 Column = 2）：访问顺序为 Col 2 → 3 → 4 → 5 → 6 → 7 → 0 → 1（wrap 回到同一行开头）。

### 7.4 预取深度与 DDR 代次的关系

预取深度决定了 "内部频率" 与 "外部传输率" 的比值关系：外部传输率 = 内部核心频率 × 预取深度。DDR3 中：800 MT/s = 100 MHz × 8n。

## 八、DDR3 专题

### 8.1 DDR3 相对 DDR2 的主要改进

| 改进项 | DDR2 | DDR3 |
|--------|------|------|
| 工作电压 | 1.8V | 1.5V（↓17%） |
| 预取深度 | 4n | 8n |
| Bank 数量 | 4/8 | 8 |
| 复位引脚 | 无 | RESET# 新增 |
| ODT | 有 | 改进（支持动态 ODT） |
| ZQ 校准 | 无 | 新增（外接 240Ω 电阻校准驱动强度） |
| 差分 DQS | 可选 | 强制 |
| Write Leveling | 无 | 新增（补偿 PCB 走线 skew） |
| 起始传输率 | 400 MT/s | 800 MT/s |

### 8.2 DDR3L 低电压版

DDR3L 是 DDR3 的低电压版本（L = Low Voltage）：

| | DDR3 | DDR3L |
|--|------|-------|
| VDD | 1.5V ± 0.075V | 1.35V ± 0.067V |
| 兼容性 | 仅 1.5V | 向后兼容 1.5V |
| 功耗 | 基准 | 比 DDR3 降低约 20% |
| 应用场景 | 桌面/笔记本 | 嵌入式/超薄本 |

**i.MX 6ULL 支持**：仅支持 DDR3L x16 和 LPDDR2 x16，不支持标准 DDR3、DDR2 或 LPDDR1。

### 8.3 8-Bank 架构

DDR3 将 Bank 数量从 DDR2 的 4 个增加到 **8 个**（BA[2:0] = 3 位）。

```
DDR3 芯片内部 8-Bank 结构:

            ┌─────────────────────────────────┐
            │          DDR3 Chip              │
            │                                 │
            │  ┌────┐ ┌────┐ ┌────┐ ┌────┐   │
      BA  ──┤→│Bank│ │Bank│ │Bank│ │Bank│   │
     [2:0]  │  │ 0  │ │ 1  │ │ 2  │ │ 3  │   │
            │  └────┘ └────┘ └────┘ └────┘   │
            │  ┌────┐ ┌────┐ ┌────┐ ┌────┐   │
            │  │Bank│ │Bank│ │Bank│ │Bank│   │
            │  │ 4  │ │ 5  │ │ 6  │ │ 7  │   │
            │  └────┘ └────┘ └────┘ └────┘   │
            │      ↕ 共享数据总线 (DQ)         │
            └─────────────────────────────────┘
```

**8 Bank 的好处**：
- 更多 Bank = 更多并发机会（一个 Bank 在预充电，其他 Bank 可读写）
- 降低 Bank 冲突概率
- 支持 Bank Interleaving（Bank 交错）优化

> **参考资料**：8-Bank 架构图见 `4Gb_DDR3_merged.md` / `JESD79-3D_merged.md` 中的 Functional Description 章节。

### 8.4 ODT（On-Die Termination，片上终端匹配）

**背景**：高速信号传输中，如果传输线终端不匹配，会产生信号反射，导致数据错误。

**传统方案**：在 PCB 上安装终端匹配电阻（外部电阻）。

**ODT 方案**：将匹配电阻集成到 DDR 芯片内部，通过 ODT 引脚动态控制。读写操作时，ODT 根据需要进行开关，减少信号反射。

```
ODT 工作原理:

SoC ────────DQ/DQS 总线────────→ DDR 芯片
                                │
                    ┌─── 240Ω ──┤
                    │           │  ← ODT 电阻网络（片上集成）
                    └─── 120Ω ──┤
                    │           │
                    └─── 60Ω ───┤
                                │
                              VDDQ / VSS

ODT 动态控制:
  读操作: ODT 关闭（或低阻）  ← 避免干扰芯片输出
  写操作: ODT 开启（匹配阻）  ← 吸收信号，防止反射
```

**ODT 配置**：
- MR1.A6：动态 ODT 控制
- MR2.A5-A7：动态 ODT 电阻值（40Ω / 60Ω / 120Ω）

**优势**：
- 减少 PCB 外部元件
- 可以根据读/写操作动态开关 ODT
- 降低功耗（不需要时关闭）

> **参考资料**：ODT 时序图见 `JESD79-3D_merged.md` 中的 Figure 74-87（Dynamic ODT Timings）。

### 8.5 ZQ 校准

ZQ 校准是 DDR3 新增功能，用于校准输出驱动强度和 ODT 电阻值。DDR 芯片的 ZQ 引脚外接 240Ω ± 1% 精密电阻到 VSS，通过测量该参考电阻来校准内部驱动和 ODT 值。

```
ZQ 校准硬件连接:

              DDR3 芯片
            ┌───────────┐
            │           │
      ZQ  ──┤  校准电路  │
            │           │
            └─────┬─────┘
                  │
              240Ω ± 1%    ← 外部精密参考电阻
                  │
                VSS (GND)

校准过程:
  1. 芯片内部测量 ZQ 引脚上的参考电阻
  2. 根据测量结果调整驱动强度 (RON)
  3. 根据测量结果调整 ODT 电阻值
  4. 校准结果存入内部寄存器
```

**校准类型**：

| 类型 | 触发时机 | 持续时间 |
|------|---------|---------|
| ZQCL（Long Calibration） | 上电初始化、退出 Self Refresh | tZQinit ≈ 512 周期 |
| ZQCS（Short Calibration） | 周期性执行 | tZQoper ≈ 64 周期 |

**i.MX 6ULL MMDC 的 ZQ 支持**：
- 硬件自动 ZQ Short（周期性）
- 硬件自动 ZQ Long（退出 Self Refresh 时）
- 软件手动 ZQ INIT（初始化时）
- IO 焊盘驱动强度校准

> **参考资料**：ZQ 校准流程见 `4Gb_DDR3_merged.md` 中的 ZQ Calibration 章节、`JESD79-3D_merged.md` 中的 ZQ 相关描述。

### 8.6 差分 DQS / 差分时钟

DDR3 强制使用差分信号：

| 信号 | 差分对 | 作用 |
|------|--------|------|
| 时钟 | CK / CK# | 命令/地址采样基准 |
| 数据选通 | DQS / DQS# | 数据采样基准 |

**差分信号的优势**：
- 抗共模噪声
- 更精确的时序定位
- 支持更高频率

### 8.7 复位引脚（RESET#）

DDR3 新增的 RESET# 引脚：

- **功能**：将 DDR 芯片置于低功耗复位状态
- **有效电平**：低有效
- **上电要求**：JEDEC 规定上电期间 RESET# 必须保持低电平，直到 VDD 稳定

**i.MX 6ULL 要求**：必须通过下拉电阻将 DRAM_RESET 信号接地，以确保上电期间保持低电平。

### 8.8 写均衡（Write Leveling）

**背景**：在多颗粒 DIMM 设计中，由于 PCB 走线长度不同，时钟信号到达各颗粒的时间存在差异（Clock Skew）。

**解决方案**：Write Leveling 允许每个 DDR 芯片动态调整 DQS 相对于 CK 的延迟，补偿走线差异。

```
Write Leveling 原理（补偿 CK 到达不同芯片的时间差）:

SoC 发出的 CK 信号:
  CK ───┬───── t1 ────→ [Chip 0]  ← 走线短，到达早
        │
        └───── t2 ────→ [Chip 1]  ← 走线长，到达晚

Write Leveling 校准前:
  Chip 0: DQS ──┐  ← DQS 与本地 CK 对齐
  Chip 1: DQS ──┐  ← DQS 与本地 CK 对齐，但相对 SoC 有 skew ❌

Write Leveling 校准后:
  Chip 0: DQS ────┐  ← 延迟 DQS，补偿 t1
  Chip 1: DQS ──┐  ← DQS 不变
  两个芯片的 DQS 在 SoC 端对齐 ✓
```

由于 PCB 上 CS0 和 CS1 的走线长度不同，i.MX 6ULL 还提供了**地址镜像**（Address Mirroring）功能，交换 CS0 和 CS1 的 BA 和部分地址线，简化 PCB 布线。

**i.MX 6ULL MMDC 支持**：Write Leveling Calibration，自动调整写 DQS 与 CK 的对齐。

> **参考资料**：Write Leveling 时序见 `JESD79-3D_merged.md` 中的 Write Leveling 章节（Figure 42-44）、`IMX6ULL_section35_1_4.md` 中的 Write Leveling Calibration 描述。

### 8.9 多用途寄存器（MPR）

MPR（Multipurpose Register）是 DDR3 中的特殊寄存器，用于：

- 读取预设数据模式（用于训练和调试）
- Self Refresh 温度监控

通过 MR3 配置 MPR 功能。

## 九、i.MX 6ULL MMDC 控制器

### 9.1 MMDC 架构：Core + PHY

![MMDC block diagram](https://cdn-mineru.openxlab.org.cn/result/2026-04-22/6dad159b-7fa4-4cd5-9a0a-2f4207cb43bf/38b044e75ed1b59d806a215d31c9ae0ade0dbf737840b05492e7c8cf9170bdb3.jpg)

- **MMDC_CORE**：通过 AXI 接口与系统通信，生成 DDR 命令、执行命令优化（重排序、Bank 预测）、管理读写数据路径。
- **MMDC_PHY**：负责时序调整，使用特殊校准机制确保高达 400MHz 时钟速率下的数据捕获余量。

### 9.2 AXI 接口与请求队列

| 特性 | 值 |
|------|-----|
| 总线协议 | AXI |
| 传输位宽 | 8 / 16 / 64 bit |
| 运行频率 | 最高 400MHz |
| 突发长度 | 最高 16 |
| 突发类型 | WRAP / INCR / FIXED |
| AXI ID 位宽 | 16 bit |

**请求队列**：
- 写请求 FIFO：8 条目
- 读请求 FIFO：16 条目

### 9.3 重排序机制与 Bank 预测

MMDC 内置重排序机制，根据动态评分（Dynamic Scoring）选择最优的 DDR 访问顺序。例如：AXI 主设备按 Bank0 → Bank3 → Bank0 → Bank1 顺序请求，MMDC 会将相同 Bank 的读写合并，减少预充电/激活次数。

**Bank 预测**：通过深度请求队列，预测下一个可能访问的 Bank，提前发出预充电/激活命令。

### 9.4 Page Hit / Page Miss 优化

| 情况 | 说明 | 优化 |
|------|------|------|
| Page Hit | 目标 Row 已在行缓冲区中 | 直接发送 READ/WRITE，跳过 ACT |
| Page Miss | 目标 Row 不在行缓冲区中 | 需要先 ACT，再 READ/WRITE |
| Page Close | 目标 Bank 空闲 | ACT → READ/WRITE |

MMDC 自动跟踪打开的内存页面，最大化 Page Hit 率。

### 9.5 校准机制

MMDC PHY 支持多种校准（可硬件自动或软件手动）：

#### ZQ 校准

| 类型 | 触发方式 | 说明 |
|------|---------|------|
| ZQ Short | 硬件自动（周期性） | 短期校准，维持驱动强度精度 |
| ZQ Long | 硬件自动（退出 Self Refresh） | 长期校准，完全重新校准 |
| ZQ INIT | 软件手动 | 初始化时执行 |
| IO 焊盘校准 | 软件/硬件 | 校准 DDR IO 驱动强度 |

#### 读/写数据校准

| 校准类型 | 说明 |
|---------|------|
| 读数据校准 | 调整读 DQS 与读数据字节的对齐 |
| 读 DQS 门控校准 | 调整 DQS 门控与读前导窗口的对齐（仅 DDR3） |
| 写数据校准 | 调整写 DQS 与写数据字节的对齐 |
| Write Leveling | 调整写 DQS 与 CK 差分时钟的对齐 |
| 读精细调优 | 每个读数据位最多 7 个延迟线单位 |
| 写精细调优 | 每个写数据位最多 3 个延迟线单位 |

#### 附加精细调优

- DDR 时钟延迟调整
- DDR 时钟占空比调整
- DQS 占空比调整

### 9.6 省电模式

#### Self-Refresh（自刷新）

DDR 芯片在 Self Refresh 模式下**自行管理刷新操作**，MMDC 可以关闭 DDR 时钟。

**进入方式**：
1. **LPMD（Low Power Mode）**：硬件/软件握手
2. **DVFS（动态电压频率调节）**：硬件/软件握手
3. **自动进入**：配置空闲周期数后自动进入

**退出流程**：
1. 打开 MMDC 工作时钟
2. 驱动 CK 时钟到 DDR
3. 满足 tCKSRX 后，断言 CKE
4. 解除 LPACK/DVACK
5. 满足 tXS 后，发出 REF 命令
6. 如果 ZQ 校准启用，执行 ZQ Long
7. 满足 tDLLK 后，恢复正常操作

#### Power Down（掉电模式）

比 Self Refresh 更轻量级的省电模式：

| 类型 | 说明 |
|------|------|
| Precharge Power Down | 所有 Bank 预充电后进入掉电 |
| Active Power Down | 保持 Row 激活状态下进入掉电 |
| Fast Exit | 快速退出（功耗略高，退出快） |
| Slow Exit | 慢速退出（功耗更低，退出慢） |

**自动进入配置**：ESDPDC 寄存器中的 PWDT_0/PWDT_1 定义空闲周期数。
